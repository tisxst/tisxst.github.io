在上一个笔记中，我们简单了解了关于所有权的问题。在笔记结尾，我们针对一个值的所有权进行了几次的尝试，但是无法避免的每次所有权转移都需要重新对变量进行操作，有没有什么办法既不变更所有权，又能读取数据？

Rust 对于这种情况当然也有足够的考虑，那就是引用（reference）与借用（borrow）。

1 - Reference & borrow

首先看下面这段代码

这里我们在定义函数 calculate_length 时使用了 s: &String，这个 & 为引用符号，相当于形式参数 s 引用了一个实际的 String 变量。我们在实际调用这个函数的时候传入的也是 &s1，因此所有权并没有转移，因此引用这个动作，也叫做 borrow（借用）。

这个玩意在 C 语言指针中用的很多，而且 C 语言还有个 *，在 Rust 中也有，后续再记录。

2 - Change while borrowing

当我们想通过引用变更堆中的 mutable 值时要怎么操作呢？

需要注意的是，这种变更必须要确保：

  1. 变量本身为 mutable
  2. 传入方法 change 的实参必须要显式标注 &mut
  3. 创建方法 change 时必须显式规定传入的形参为 &mut

缺一不可，否则编译器就会报错。

4 - Dangle reference

在 C 语言中，有一种很危险的指针操作叫作悬垂引用，也就是由于代码逻辑问题，使得指针指向了一块已经被回收的内存空间，而对这块被回收的内存空间的操作会引起危险。

Rust 的所有权设置很好的避免了悬垂引用的出现

本身 dangle 返回了一个指向 String 的引用，不管这块内容是否可边，Rust 都会直接阻止这种代码出现。

如果想让代码正常运行，那返回值可以是 s 本身，因为这就是所有权转移了，而不是借用。Rust 对于借用的原则就是，你要借的变量一定是一个有效值。
