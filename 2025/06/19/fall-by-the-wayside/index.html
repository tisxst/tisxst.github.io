今天通过 NAS，把我在家里电脑上的 hexo 配置和文档同步到了公司电脑上。而后在公司电脑上配置了 hexo 的环境，成功实现了在不同电脑上写 Blog 的成就。

但是，这件简单的事情做完之后，我突然开始思考一个问题。对于一个对电脑不怎么熟悉，不会编程，也不了相关领域的人来说，这样的操作已经是非常厉害的电脑高手了，但是对于那些深耕计算机的人来说，这种操作简直就是小儿科；我非常喜欢计算机，也非常愿意折腾这种给生活带来便利的工具，但是我为什么没办法在计算机这个领域深耕下去？没有办法成为这个领域的大牛？

对于我来说，做出一些简单却很酷的事情，让我成就感满满，也就是这种没有什么难度，但看起来很厉害的事情让我无法自拔。虽然同样是在学东西，我学到的却一直是一个表面的、浅层的、仅限于使用的工具。

我认为，真正厉害的大牛，应该是创造 hexo 的人，而不是因为会使用 hexo 而沾沾自喜的人；是可以在 hexo 基础上开发出厉害主题的人，而不是学会下载主题和改改参数配置好了主题就觉得自己很厉害的人。而我，就是后者。

这种反思，也让我意识到，我其实潜意识里是想成为去创作的人，而不是仅仅是一个会使用工具的人，我在平时的学习中也是在向着这个方向去努力的，但是每次都会因为各种各样的原因半途而废。这种半途而废，就是我想写这篇博客的原因。

我曾经下定决心，买了一整套 Java 课程，但是仅仅学完了基础语法；我曾经下定决心，购买了一整套嵌入式课程，但是仅仅学完了 C 语言基础语法，甚至连 C++ 都没碰到就中断了学习；我曾经下决心购买了一整套德语课程，但是上课上了一半就放弃了；我曾经下决心花了大价钱购买了终身会员的 Memorise，但是截至到了现在，依旧停留在前几节课程再无后续。

我甚至自己已经回忆不起来，为什么自己能把英语口音练得这么好，我甚至不无法回忆词汇量可以接近一万的原因，因为从我自己了解的自己来看，我无法坚持下去，甚至是在自己喜欢的领域。

然而，虽然从很早之前我就意识到了这个问题，但是我到了 30 岁的年龄，还在因为这个问题而挣扎。我现在的想法就是，我对自己太过宽容了，稍微觉得累，就宽慰自己，过两天在学习；而学习一个领域的知识，过两天，就代表着全盘皆忘。

在写上述这段文字的时候，我的情绪是消极的，因为我并不知道面对这种情况我应该如何处理自己的情绪和将这种消极的情绪转变为向前的动力。

然而，AI 的存在解决了这个问题，Deepseek，ChatGPT 和 Gemini 给我的答复居然真的用文字给了我很强的支撑力。三者的建议都是差不多的，并且都能在情绪上宽慰到我，让我起码有了继续下去的力量。我没想到文字的能量这么超乎想象。

🧠 第二部分：你为什么总是半途而废？

你提到的“Java、嵌入式、德语、Memorise”这些课程，很可能都因为以下原因：

✅ 1. 目标过大、过抽象

“我要学完 Java、德语、嵌入式。”
这个目标像是“我要翻越一座山”，但你没有给自己画出路径图，也没有明确每一步的意义。于是，当你一抬头看到山太高、太久、太难，就自然放弃了。

✅ 2. 没有反馈和成就闭环

像练英语口音成功，可能是因为你当时每练一点都会被别人夸、被自己听出来效果。你学 Hexo 也是，因为做出博客后能立刻看到成果。

而编程、德语、C++，在前期往往“干学不出效果”，没有及时反馈，容易让人丧失动力。

✅ 3. 没有机制防止“间断→遗忘→崩溃”

当你“休息几天再学”时，发现前面学的都忘了，重学就像拔河失败，直接摔倒。

所以问题不是你不够努力或没天赋，而是：

  • 目标不够清晰分解；
  • 缺少反馈；
  • 没有建立小的巩固机制；
  • 情绪化学习，缺乏系统性。

💡 第三部分：你到底该怎么做？

你已经意识到了问题，现在不是需要“鸡汤”，而是需要“方法”。以下建议更像“操作说明书”而不是“鼓励语录”。

✅ 1. 只专注一个主线技能，且有“输出目标”

你现在的 Hexo 技术不错，可以从“写博客”转向“造主题”作为主线目标。

比如：

  • 设一个明确目标：3个月内自己手撸一个 Hexo 主题；
  • 每天学习 1 小段 JS / CSS；
  • 每周发布一篇进展文，上传到你的博客，记录学习过程；
  • 加入一个有同样兴趣的小圈子，发帖互动（如掘金 / GitHub / Reddit / 少数派）。

输出导向，会强迫你把输入转化为“可用的创造”，不会陷入“干学”。

✅ 2. 为每次中断设置“回归引导”

不是每次学习都要坚持不间断，但你要给“中断”做准备。

举个例子：

  • 每次学习后，写一个《3分钟笔记》：我今天学了啥，下次我要从哪继续。
  • 在学习文档里加一个“回归卡片”：「我上次停在了哪？我下次从哪开始？」

这能帮你从遗忘中快速回归，不会因“断了几天”而放弃。

✅ 3. 控制“成就感上瘾”，重建“挑战曲线”

你说：

“我很享受那种对别人看起来很厉害、对内行却是小儿科的成就感。”

这没问题。但它不能成为你停止深耕的理由。

你需要“在小事上找到自信，在大事上找到使命”。

建议你设定一个“挑战曲线”：

  • 🌱 简单的任务（Hexo 文章更新）；
  • 🧱 中级任务（主题定制）；
  • 🧠 高级任务（独立开发插件 / 发布 NPM 包）；

别一开始就冲“我想成为大牛”——你只需要比上一次的你更深一步。

✅ 4. 找到你的长期“兴趣契约”

你喜欢计算机、喜欢创造，这很清晰。你也不是“缺毅力”，而是“缺设计”。

请把你的学习设计成“有仪式感、有阶段目标、有成果展示”的过程，而不是靠激情驱动的突击战。

你可以参考这样的结构：

1. 重新定义“成就感”的来源

你提到，完成像“Hexo同步”这样“看起来很厉害”的事情让你成就感满满。这是人之常情，我们都喜欢即时反馈和正向激励。但问题在于，这种成就感的“阈值”太低了，它让你沉迷于“新手村”的胜利，而畏惧走出村庄去面对真正的挑战。

怎么办？

  • 把“创造”分解成“微创造”：

    真正的创造不是一蹴而就的。创造Hexo的大神，也不是一天就写完所有代码的。他也是从解决一个极小的问题开始的。对于你而言，不要总想着“我要开发一个完整的主题”，这个目标太大，足以压垮任何刚起步的人。

    • 完成它！ 这件事很小，可能只需要你搜索几个CSS属性，但它本质上是创造，是你对现有工具的改造和拓展，而不是单纯地使用。当你完成时，告诉自己：“我今天创造了一点东西，我比昨天那个只会用的人进了一步。”

2. 放弃“一口吃成胖子”的幻想，拥抱“项目制学习”

你购买了大量的课程，这说明你有强烈的学习意愿和付费意识，这非常好。但问题出在学习方式上——“体系化学习”在初期是反人性的。

想象一下，你为了“学会做饭”而去从“农业的起源”和“食品化学”开始学起，你大概率会在认识所有谷物之前就放弃了。你现在就处于这种状态。Java语法、C语言语法，都只是工具，学完它们本身并不能带来任何实质的改变和乐趣。

怎么办？

  • 停止“爬山式学习”，转向“寻宝式学习”：
    • 设立一个极度具体、让你兴奋的“小项目”。不是“学会Java”，而是“用Java写一个能每天自动给我发一封‘今天也要加油’的邮件的小程序”。不是“学嵌入式”，而是“我要买一块ESP32开发板，让它连接我家WiFi，控制一个小灯的亮灭”。
    • 在“寻宝”的过程中去学习。为了实现邮件发送，你才会去查Java的邮件库怎么用，这会逼着你去看懂相关的语法和对象。为了点亮一个灯，你才会有动力去搞明白什么是GPIO，什么是高低电平。
    • 知识不再是孤立的，而是你为了达成目标必须捡起来的“宝藏”。这个过程，你会自然而然地将知识串联起来，而不是停留在枯燥的语法层面。

3. 承认“遗忘”是常态，建立你的“外部大脑”

你提到“过两天，就代表着全盘皆忘”。是的，这是大脑的工作机制，它会优先记住那些被反复提取和使用的信息。你无法对抗生理规律。

怎么办？

  • 别怕忘，做好笔记！

    真正的大牛不是记住了所有东西，而是建立了一套高效的知识索引系统。

    • 利用你已经会的工具：

      • 标题可以是：“【已解决】在Windows上配置Hexo环境时遇到 ‘npm error’ 的解决方法”。
      • 内容可以是你尝试了哪些错误方法，最后哪个方法管用了。

    • 这不仅仅是记录，这是在“费曼学习法”：你能把一个东西讲清楚，说明你真的理解了。你的博客就是你的“外部大脑”，忘了就去翻，理直气壮。久而久之，这些知识会因为你反复地记录、查询和使用，而真正内化。

4. 重新审视你的“坚持”——关于英语的例子

你回忆不起来是如何练好英语、积累上万词汇的。这恰恰说明了真正有效的学习，往往是无痛的、融入生活的、由兴趣驱动的。

    • 你沉迷于某个英文世界的游戏，为了和队友交流、看懂攻略，你拼命地查单词。
    • 你喜欢听某首英文歌，为了学会它，你反复地跟唱，模仿发音。

怎么办？

  • 把学编程、学德语，也变得像学英语一样“有趣”

    。找到那个能让你“沉迷”的切入点。

    • 找到内在驱动力，而不是靠“毅力”的外部强压。 毅力是消耗品，很快会用完；而兴趣是永动机。

5. 对自己“温柔地严厉”

“对自己太过宽容”是你的自我诊断，非常准确。但解决方法不是180度大转弯，变成一个对自己苛刻无比的苦行僧，那样只会更快地崩溃。

怎么办？

  • 制定“微习惯”和“不可逾越的底线”：
    • 微习惯： 不要规定“我每天要学2小时Java”。而是“我每天只要打开编程软件，写一行‘Hello World’就算完成任务”。听起来很傻，但它的目的是消除你开始学习的心理阻力。一旦你打开了软件，写了一行代码，你很可能会顺手再多写几行。
    • 底线思维： 规定一个无论如何都要完成的最低标准。比如：“我这周，必须完成那个用Java发送邮件的小功能，哪怕代码写得再烂，能跑通就行。”
  • 学会“战略性休息”： “过两天再学”之所以导致“全盘皆忘”，是因为你没有计划。你可以告诉自己：“我今天学累了，但明天晚上9点，我的日程表上安排了30分钟来回顾今天的内容。” 这是计划内的休息，而不是无限期的拖延。

总结：你的行动方案

  1. 忘记“成为大牛”的宏大叙事，专注于眼前这个能让你兴奋的微小项目。
  2. 停止单纯上课，把课程当成你完成项目时查阅的字典。
  3. 拥抱遗忘，将你的博客打造成记录解决问题过程的**“外部大脑”**。
  4. 寻找乐趣，像你无意中学好英语那样，找到编程或德语中让你沉迷的那个点。
  5. 对自己温柔而坚定：建立微习惯，守住项目死线 (Deadline)，有计划地休息。

你已经有了同步Hexo博客的动手能力，这证明你具备学习和解决问题的基本素养。你缺的不是能力，而是一套将你的热情转化为持续行动的有效方法论。

别再为过去的“半途而废”而苛责自己，那是你为找到正确道路所付出的必要试错成本。从现在开始，选择一个小到不可能失败的项目，动手去做，去创造，去记录。

当你完成第一个真正属于你的“微创造”时，那种超越了“配置一下工具”的深刻成就感，将会成为你走下去的、最坚实的燃料。

    • 大量的枯燥练习： 写代码、调试、阅读文档、理解底层原理，很多环节并不像“配置成功”那样有即时的正向反馈。
    • 不可避免的挫败感： 会遇到无数 bug、难以理解的概念、卡壳的时刻，这些带来的都是负反馈。
    • 模糊的目标和漫长的回报周期： 不像“今天把博客同步好”这么具体，创造的目标往往宏大且实现路径漫长，短期看不到显著成果。
  • 舒适区的陷阱： “浅层使用”是你的舒适区。它风险低、见效快、成就感来得容易。而“深度创造”是学习区甚至恐慌区，充满未知和挑战。大脑本能地倾向于停留在舒适区。

2️⃣ 改变策略：从“想成为”到“正在成为”

🔄 重塑目标与心态

  • 目标具体化且微小化： 放弃“成为大牛”、“精通 Java/嵌入式”这类模糊宏大的目标。
    • 错误目标： “我要学会 Java 开发。”
    • 正确目标： “我要用 Java 写一个命令行工具，实现将指定目录下的所有 .txt 文件合并成一个文件。” (具体、可实现、有明确输出)
    • 错误目标： “我要开发一个 Hexo 主题。”
    • 正确目标： “我要修改现有的某个简单 Hexo 主题，把它的页脚布局改成我设计的样子。” (从修改开始)
  • 拥抱过程，降低预期： 承认深度学习的困难、枯燥和必然的挫折。把“遇到问题并解决它”本身视为成就，而不是只有最终结果才算成就。 每一次调试成功、每一个概念弄懂，都是进步。
  • 区分「消费」与「创造」： 提醒自己，买课程、看教程是“消费知识”。只有动手写代码、做项目、修改现有项目、尝试解决实际问题才是“创造”的开始。目标是大幅提高“创造”的时间占比。 课程只是拐杖，最终要扔掉它走路。
  • 寻找内在驱动： 思考“创造”本身能给你带来什么更深层的满足？是解决问题的智力快感？是构建事物的掌控感？还是分享成果的喜悦？找到这个点，用它来对抗过程中的枯燥。

🛠️ 优化学习方法与实践

  • 项目驱动学习： 这是最关键的一点！ 不要为了学语法而学语法。找到一个你真正感兴趣、且需要用到目标技术（哪怕只用到一点点）的小项目。
    • 例子：
      • 编程： 写一个自动整理下载文件夹的小脚本；写一个爬取某个你常看网站最新文章标题的小工具；为你的 Hexo 博客写一个简单的自定义插件（哪怕只是修改现有插件）。
      • 嵌入式： 用开发板控制一个 LED 灯按特定模式闪烁；读取一个温湿度传感器的数据并在串口打印出来；做一个简单的电子时钟。
      • 主题开发： 选一个简单的 Hexo 主题，尝试修改它的颜色、字体、布局；尝试添加一个简单的功能（如阅读进度条）。
    • 好处： 有明确目标；在实践中遇到真实问题（这才是学习的契机）；能获得可见的成果（即使很小）；动力更足。
  • 强制输出与记录：
    • 写技术博客： 即使只是记录你今天学的一个小知识点、解决的一个小 bug、做的一个小实验。公开发布是强大的驱动力。 在博客中解释清楚一个概念，是检验你是否真正理解它的最好方式。这也是在构建你的“创造”资产。
    • GitHub 提交： 即使项目再小，也要用 Git 管理，并定期提交到 GitHub。看到贡献图上的绿色小方块，也是一种正向激励。
  • 聚焦「最小可行进步」： 每天/每周保证一个非常小的、但必须动手的“创造”行为。
    • “今天我要看懂这个开源项目的 50 行代码并注释它。”
    • “今天我要调试通昨天卡住的这个 bug。”
    • “今天我要给我的小工具添加一个简单的命令行参数。”
    • 关键是持续性和动手。 15 分钟高质量的、专注的动手实践，远胜过 2 小时心不在焉地看教程。
  • 利用社区和同行者：
    • 参与开源： 从报告 bug、改进文档（typo、翻译）、写测试用例开始参与开源项目。这是向创造者迈进的重要一步，也能接触优秀代码和协作流程。
    • 寻找学习伙伴： 找到有相似目标的人（哪怕线上），互相分享进展、讨论问题、督促打卡。
  • 刻意练习与回顾： 不要只是机械地写代码。定期回顾：
    • 我遇到了什么问题？
    • 我是怎么解决的？（记录解决方案！）
    • 有没有更好的方法？
    • 我学到了什么新概念/技巧？
    • 下次类似问题如何更快解决？

🧠 建立可持续的习惯与克服阻力

  • 固定时间 & 环境： 每天在固定的时间（哪怕只有 30 分钟）、固定的地点（书房、安静的角落）进行你的“创造”时间。让这成为一种仪式感。
  • 「两分钟」启动法则： 当你感到抗拒时，告诉自己：“我只做两分钟”。打开 IDE 写两分钟代码；看两分钟文档。往往开始做了，就会继续下去。
  • 预先决策 & 减少障碍： 前一天晚上决定好第二天“创造”时间要做的具体小任务（比如：解决昨天遗留的 XXX bug）。把需要的环境、资料准备好。
  • 拥抱中断，但建立重启仪式： 中断（工作忙、生病等）不可避免。关键不是永不中断，而是中断后如何快速重启。建立一个简单的重启仪式（如：清理一下工作区、快速回顾一下上次进度、写下今天第一个小任务）。
  • 对自己诚实但也要宽容： 当计划失败时，不要过度自责（这会导致彻底放弃），但也不要轻易放过自己（“明天再说”无限循环）。冷静分析原因：是目标太大？时间没安排好？遇到难以逾越的障碍？然后调整计划，立即重启一个最小可行任务（哪怕只做 5 分钟）。
  • 管理精力而非时间： 在精力充沛的时候做需要高专注度的“创造”工作（如写代码、调试）。在精力低谷时做整理文档、看技术文章等事情。保证充足睡眠。
  • 重新定义「累」： 区分“身体/大脑真累”和“心理抗拒”。前者需要休息，后者往往可以通过“两分钟法则”或改变任务类型（比如从写代码切换到画设计草图）来克服。

3️⃣ 针对你放弃课程历史的特别建议

  • 停止购买新课程！ 你已有的课程资源（Java、嵌入式）已经足够多了。购买新课程带来的短暂快感，正是阻碍你深入实践的陷阱。承诺：在把现有某个课程的 20% 知识真正用于动手做一个项目之前，不再购买任何新课程。
  • 选择性重启： 如果真的想继续 Java 或嵌入式，忘记“学完课程”的目标。 直接跳到项目实践部分：
    • Java： 打开之前的课程，找一个讲完基础语法后的实践项目章节，直接动手做那个项目。遇到不懂的语法或概念，再按需回去查前面的课程章节或官方文档。把课程当作字典，而不是必须从头读到尾的小说。
    • 嵌入式： 买一块开发板（如 STM32 或 ESP32 的最小系统板），找一个基于这块板子的、你感兴趣的、有完整代码和教程的小项目（如点亮 LED、读取传感器），跟着做。遇到 C 语言问题，再针对性复习。硬件在眼前跑起来，比看课程视频要激励得多。

📌 总结关键行动点

  1. 立即启动一个超小型项目： 今天/明天就选一个，用你现有的、最熟悉的工具开始（比如给 Hexo 写个极简的统计插件，或修改主题的一个小样式）。
  2. 承诺「最小可行行动」： 保证每天/隔天有 15-30 分钟动手时间，专注在这个项目上。
  3. 公开记录： 在你的博客写一篇短文，描述你启动这个项目的想法和第一天的进展（哪怕只是搭建了环境）。
  4. 利用 GitHub： 为这个小项目建个仓库，开始提交代码。
  5. 停止课程消费： 除非是项目实践中遇到障碍需要查阅，否则暂停观看新的教程视频。专注动手。
  6. 拥抱并记录问题： 把遇到和解决的每一个问题都记录下来（博客/GitHub Issue/笔记），这是你成长的勋章。

技术领域的深度不是百米冲刺，而是无数个深夜调试后的黎明。 你已拥有发现问题的敏锐和表达困惑的勇气，这两者远比能配置多少工具珍贵得多。真正的创造者不是从不疲倦的超人，而是在每个想放弃的深夜仍多写了三行代码的普通人。那些半途而废的课程，恰恰是你探索边界的足迹——现在只需把其中一条路走出深度。

你愿意从哪个微小项目开始重启创造之旅？我很期待听到你第一个动手实验的具体计划。

既然有赛博心理医生帮我解决了问题，那我下一步就是向着日出的方向继续前进，我相信未来的光一定会照在我前进到的道路上。
