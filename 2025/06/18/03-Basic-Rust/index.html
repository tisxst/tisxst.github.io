这里介绍了和其他变成语言一样的 Rust 基础概念，从变量开始。

1 - Variables

Rust 的一个很重要的机制就是”安全，安全，还是他妈的安全“。这从语言设计的方方面面得以体现。

比如 Rust 的变量默认为不可重新赋值的变量，也就是 immutable，所以更应该称之为常量。

如果你使用 IDE，第四行会直接提示你有错误。然而，大家都知道变量的存在是有其存在意义的。所以为了创建一个可变变量，需要使用 mut 关键字。

Rust 除了这种「不可变的变量」之外，依旧有自己的「常量」。不管怎么样，常量都不会改变，所以给常量加 mut 是不管用的。因此声明一个常量应使用 const 关键字。这个内容后续再讲。

Rust 常量的命名原则是大写字母加下划线分割，类似 DONT_KNOW_WHY。

但是 Rust 允许重新 let 同名变量

这种情况下，我们可以在不同的场合使用相同的变量名，但是变量类型可以不同。由于 let 是重新创建了一个新变量，因此老变量就被覆盖了。

2 - Data type

上一篇笔记中我们记录了数值类型变量：i32, u32, i64, u64。Rust 需要在编译时知道所有变量的类型。因此当一个变量存在多个可能的类型时，编译会报错。

如进行两个不相同的数据类型之间的的运算，编译器会报错。

2.1 - Integer

除了上述这四种数据类型之外，还有其他长度大小的数值型变量，创建一个指定类型的变量需要使用 :

Length Signed Unsigned
8 bit i8 u8
16 bit i16 u16
32 bit i32 u32
64 bit i64 u64
128 bit i128 u128
arch isize usize

arch 是 architecture 的缩写，也就是说这个类型的变量大小取决于计算机的架构，如果是 32 位的架构，就是 32 位大小，64 位架构就是 64 位大小。

这里有无符号位和 C 语言的逻辑基本是一样的：所有数据都是用补码进行储存。

2.2 - Float

与正数不同，所有的浮点数都是 signed，而且只有两种类型的浮点 f32 与 f64。Rust 的默认浮点类型是 f64，这是因为在现代的计算机处理器上运行速度几乎没差别。

2.2 - Boolean

与其他的编程语言一样，包含 true 与 false。

2.3 - Char

字符类型数据，使用单引号把一个字符括起来，这个字符可以是任何字符，甚至可以是 emoji。

2.4 - Tuple and array

元组与数组都是数据集合，元组创建后不可以更改长度，元组的符号是小括号 ()

如果要读取元组内的值，可以使用模式匹配，对元组元素进行结构（Destructuring）。

这里的实际操作，是将一个复杂数据结构内部的成员提取出来，并赋值给多个变量；简单地说由于元组 tup 有三个元素，并且 (x, y, z) 也是三个元素，所以二者的结构是匹配的，所以 Rust 将元组成员依次赋值给了 x, y, z 三个变量。

当然，也可以通过传统的访问元素索引位置获取元素值。

Rust 中的数组与其他语言也不同，它是同一个类型数据的集合，而且创建后无法变更长度，数组的符号是中括号 []。一个可以变更长短的数组是另一个数据类型，叫 vector。正儿八经的翻译应该是向量，但是它表达的涵义就是一个可变长度的数组。
